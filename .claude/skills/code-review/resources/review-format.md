# Review Format

How to format code review feedback for da_vinci.

## Review Structure

### Start Positive
- Acknowledge good work
- Call out well-done aspects
- Build trust before critique

### Be Specific
- Reference exact file and line numbers
- Quote the problematic code
- Explain what should change

### Explain Why
- Don't just say "this is wrong"
- Explain the reasoning
- Link to standards or docs if applicable

### Offer Solutions
- Suggest specific improvements
- Provide alternatives when possible
- Show how to fix the issue

## Feedback Levels

### Must Fix (Blocking)
- Violations of code standards
- Security issues
- Breaking changes without justification
- Missing tests for new code

### Should Fix (Non-Blocking)
- Code clarity improvements
- Performance optimizations
- Better error messages
- Additional test coverage

### Nice to Have
- Style preferences
- Alternative approaches
- Future improvements

## Tone Guidelines

### Do
- Be respectful and professional
- Focus on the code, not the person
- Ask questions to understand intent
- Offer to discuss complex topics

### Don't
- Use sarcasm or mockery
- Make assumptions about skill level
- Be vague or unclear
- Nitpick trivial style issues
